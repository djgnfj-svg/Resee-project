# Portfolio 문서 사용법

## 📁 파일 구조

```
docs/portfolio/
├── README.md                           # 전체 가이드
├── 이력서_원문_최종.md                 # ⭐ 이력서에 복사할 최종 버전
├── 이력서_원문_간결버전.md             # 더 짧은 버전 (선택사항)
├── 이력서_원문.md                      # 상세 버전 (참고용)
├── 사용법.md                           # 이 파일
│
├── [핵심 구현 - 블로그 포스트 4개]
├── 01-복습-스케줄링-자동화.md
├── 02-JWT-인증-시스템.md
├── 03-AI-콘텐츠-검증.md
├── 04-Docker-CICD.md
│
└── [트러블슈팅 - 블로그 포스트 4개]
    ├── 05-N+1-쿼리-최적화.md
    ├── 06-이메일-토큰-보안.md
    ├── 07-Celery-시작-순서.md
    └── 08-Gunicorn-Worker-최적화.md
```

---

## 🎯 3단계 사용법

### 1단계: 이력서 작성
```bash
# 이력서_원문_최종.md 파일 열기
cat docs/portfolio/이력서_원문_최종.md
```

**이 파일을 이력서 "프로젝트" 섹션에 복사**

**특징**:
- ✅ 한 줄로 임팩트
- ✅ 궁금하면 링크 클릭
- ✅ 성과 지표 명시
- ✅ 깔끔한 포맷

---

### 2단계: 블로그 포스트 업로드

**핵심 구현 4개**:
1. `01-복습-스케줄링-자동화.md` → 블로그 포스트
2. `02-JWT-인증-시스템.md` → 블로그 포스트
3. `03-AI-콘텐츠-검증.md` → 블로그 포스트
4. `04-Docker-CICD.md` → 블로그 포스트

**트러블슈팅 4개**:
5. `05-N+1-쿼리-최적화.md` → 블로그 포스트
6. `06-이메일-토큰-보안.md` → 블로그 포스트
7. `07-Celery-시작-순서.md` → 블로그 포스트
8. `08-Gunicorn-Worker-최적화.md` → 블로그 포스트

**각 파일은 완전한 블로그 포스트**:
- 한 줄 요약
- 프로젝트 배경
- 핵심 구현 (코드 포함)
- 성과 지표
- 테스트 코드
- 면접 대비 Q&A
- 배운 점

---

### 3단계: 이력서에 링크 업데이트

블로그 URL 업데이트:
```
https://yourblog.com/resee/01
→ https://your-actual-blog.com/resee/01-복습-스케줄링-자동화
```

**예시**:
```markdown
### 1. 에빙하우스 망각곡선 기반 자동 복습 스케줄링
Django Signals로 콘텐츠 생성 시 자동 복습 일정 생성, 구독 등급별 차등 적용 (1~180일, 8단계)
**→ [구현 과정 보기](https://velog.io/@myname/resee-01)**
```
