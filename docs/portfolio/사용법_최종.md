## 🎓 면접 준비 팁

### **Redis 캐싱 전략** (가장 강력!)
- "Cache-aside Pattern으로 API 응답 시간을 80% 단축했습니다"
- "Hit Rate 85%+를 달성해서 DB 부하를 80% 감소시켰습니다"
- "복습 목록은 TTL 1시간, 통계는 5분으로 데이터 특성에 맞게 설정했습니다"
- "복습 완료 시 캐시 무효화로 일관성을 보장합니다"

### **N+1 쿼리 최적화**
- "실제로 301개 쿼리를 3개로 줄였습니다"
- "`select_related()`로 JOIN, `prefetch_related()`로 ManyToMany"
- "복합 인덱스로 조회 성능 개선"

### **CI/CD 자동 배포**
- "deploy.sh 380줄로 완전 자동화했습니다"
- "Docker 자동 설치, Swap 메모리 추가, Health check 대기"
- "배포 시간 30분 → 5분으로 단축"

### **보안 강화**
- "SHA-256 해싱으로 DB 유출 시에도 안전합니다"
- "`secrets.compare_digest()`로 Timing Attack 방어"
- "일반 `==` 비교는 응답 시간으로 토큰 추측 가능"

### **API 문서 자동화** (2025년 핵심!)
- "drf-spectacular로 OpenAPI 3.0 스키마 자동 생성했습니다"
- "수동 문서 작성 시간을 완전히 제거했습니다 (주 3.5시간 절약)"
- "Swagger UI로 프론트엔드 협업이 200% 향상됐습니다"
- "코드 변경 시 문서가 자동으로 동기화됩니다"

### **Redis Rate Limiting** (2025년 트렌드!)
- "locmem에서 Redis로 마이그레이션해서 Rate Limit 정확도를 100%로 만들었습니다"
- "멀티 프로세스 환경에서 Worker별 독립 캐싱 문제를 해결했습니다"
- "로그인 Brute Force 공격을 정확히 차단할 수 있게 됐습니다 (IP당 5회/분)"
- "서버를 증설해도 Rate Limit이 정확히 유지됩니다 (Scale-out 준비)"
