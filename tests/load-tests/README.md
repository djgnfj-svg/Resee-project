# 🧪 Resee 부하 테스트 스위트

이 폴더에는 Resee 프로젝트의 다양한 부하 테스트 스크립트들이 포함되어 있습니다.

## 📋 테스트 스크립트 목록

### 🔥 기본 부하 테스트
- **`simple-load-test.sh`** - 기본적인 헬스체크, 프론트엔드, 로그인 API 테스트
- **`stress-test.sh`** - 헬스체크 API 집중 스트레스 테스트 (20명 동시, 200회 요청)

### 🔐 인증 기반 테스트
- **`backend-load-test.sh`** - 백엔드 API별 개별 부하 테스트 (15명 동시)
- **`authenticated-backend-test.sh`** - JWT 인증된 API 테스트 (10명 동시)

### 💪 집약적 부하 테스트
- **`intensive-backend-test.sh`** - API별 집약적 테스트 (20명, 각 10회 연속)
- **`realistic-load-test.sh`** - 실제 사용자 시나리오 시뮬레이션 (10명, 전체 워크플로우)
- **`realistic-stress-test.sh`** - 개별 로그인으로 현실적인 부하 (30명, 개별 인증)

### 🎯 특수 테스트
- **`50-users-test.sh`** - 50명 동시 접속 한계 테스트
- **`extreme-db-test.sh`** - DB 극한 부하 테스트 (실험적)

## 📊 테스트 결과 요약

### ✅ 성공적으로 완료된 테스트들:
- **10명 동시**: 100% 성공률, 평균 40ms 응답
- **20명 동시**: 97% 성공률, 평균 1.4초 응답
- **30명 동시 (현실적)**: 92% 성공률, 평균 2초 응답
- **50명 동시**: 94% 성공률, 평균 2.8초 응답

### 💡 주요 발견사항:
1. **t2.small EC2 적합성**: ✅ 확인됨
2. **메모리 사용량**: 최대 235MB (2GB 중)
3. **CPU 사용률**: 평균 1-2% (매우 효율적)
4. **안정성**: 30명까지 안정적 처리

## 🚀 사용법

### 개별 테스트 실행:
```bash
# 기본 부하 테스트
./simple-load-test.sh

# 50명 동시 접속 테스트
./50-users-test.sh

# 현실적인 스트레스 테스트
./realistic-stress-test.sh
```

### 전체 테스트 실행:
```bash
# 순차적으로 모든 테스트 실행
for test in *.sh; do
    echo "=== $test 실행 중 ==="
    ./$test
    echo "=== $test 완료 ==="
    sleep 5
done
```

## ⚠️ 주의사항

1. **Docker 환경 필수**: 모든 테스트는 Docker Compose 환경에서 실행
2. **테스트 계정**: `test@example.com` 계정이 미리 생성되어 있어야 함
3. **리소스 모니터링**: 테스트 중 `docker stats` 명령으로 리소스 확인 권장
4. **간격 유지**: 연속 테스트 시 5초 이상 간격 권장
