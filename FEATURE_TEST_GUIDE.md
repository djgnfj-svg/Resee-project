## 🎯 주요 기능 테스트 체크리스트

### 1. 사용자 인증 시스템

#### 1.1 회원가입
- [ ] http://localhost:3000/register 접속
- [ ] 새 이메일로 회원가입 (예: test3@resee.com)
- [ ] 비밀번호 규칙 확인 (최소 8자, 특수문자 포함)
- [ ] 이용약관 및 개인정보처리방침 동의
- [ ] 회원가입 완료 후 자동 로그인 확인
- [ ] 대시보드로 리다이렉트 확인

#### 1.2 로그인/로그아웃
- [ ] http://localhost:3000/login 접속
- [ ] test@resee.com / test123! 로 로그인
- [ ] "로그인 상태 유지" 체크박스 테스트
- [ ] 잘못된 비밀번호 입력 시 에러 메시지 확인
- [ ] 로그인 성공 시 대시보드 이동
- [ ] 우측 상단 프로필 메뉴에서 로그아웃
- [ ] 로그아웃 후 로그인 페이지 리다이렉트

#### 1.3 Google OAuth 로그인
- [ ] 로그인 페이지에서 "Google로 로그인" 버튼 클릭
- [ ] Google 계정 선택
- [ ] 권한 동의
- [ ] 자동 회원가입/로그인 확인

### 2. 콘텐츠 관리

#### 2.1 콘텐츠 생성
- [ ] http://localhost:3000/content 접속
- [ ] "새 콘텐츠" 버튼 클릭
- [ ] 제목 입력 (3자 이상)
- [ ] TipTap 에디터로 콘텐츠 작성
  - [ ] 굵기, 기울임, 밑줄 서식 적용
  - [ ] 목록 생성 (순서 있음/없음)
  - [ ] 코드 블록 삽입
  - [ ] 인용구 추가
  - [ ] 링크 삽입
- [ ] 우선순위 설정 (높음/보통/낮음)
- [ ] 카테고리 선택 또는 생성
- [ ] 저장 후 목록에 표시 확인

#### 2.2 카테고리 관리
- [ ] 콘텐츠 작성 시 "+ 새 카테고리" 버튼 클릭
- [ ] 카테고리 이름 입력 (예: "📚 영어학습")
- [ ] 생성 후 자동 선택 확인
- [ ] 콘텐츠 목록에서 카테고리별 필터링 테스트

#### 2.3 콘텐츠 수정/삭제
- [ ] 콘텐츠 목록에서 기존 항목 클릭
- [ ] 상세 페이지에서 "수정" 버튼 클릭
- [ ] 내용 수정 후 저장
- [ ] 삭제 버튼 클릭 및 확인 다이얼로그 테스트

### 3. 복습 시스템 (에빙하우스 망각곡선)

#### 3.1 복습 스케줄 확인
- [ ] http://localhost:3000/review 접속
- [ ] 오늘의 복습 목록 확인
- [ ] 밀린 복습 항목 표시 확인 (빨간 배지)
- [ ] 복습 간격 확인 (1일 → 3일 → 7일 → 14일 → 30일...)

#### 3.2 복습 수행
- [ ] 복습 항목 클릭
- [ ] 콘텐츠 내용 확인
- [ ] 복습 결과 선택:
  - [ ] "완벽히 기억함" → 다음 간격으로 이동
  - [ ] "부분적으로 기억함" → 현재 간격 유지
  - [ ] "잊어버렸음" → 1일로 재설정
- [ ] 복습 완료 후 목록에서 제거 확인
- [ ] 다음 복습 날짜 업데이트 확인

#### 3.3 구독 티어별 복습 제한
- [ ] FREE 티어: 최대 7일까지의 복습만 표시
- [ ] BASIC 티어: 최대 30일까지의 복습 표시
- [ ] PREMIUM 티어: 최대 60일까지의 복습 표시
- [ ] PRO 티어: 모든 복습 표시 (최대 180일)

## 🐛 알려진 이슈 및 해결 방법

### 이슈 1: 카테고리 생성 실패
- **증상**: "카테고리 생성에 실패했습니다" 에러
- **해결**: 
  1. 카테고리 이름이 중복되지 않는지 확인
  2. 특수문자 제거 후 재시도
  3. 페이지 새로고침 후 재시도

### 이슈 2: 복습 항목이 나타나지 않음
- **증상**: 오늘의 복습 페이지가 비어있음
- **해결**:
  1. 콘텐츠가 최소 1개 이상 있는지 확인
  2. 구독 티어 확인 (FREE는 7일까지만)
  3. 페이지 새로고침

### 이슈 3: AI 질문 생성 실패
- **증상**: "API 키가 설정되지 않았습니다" 에러
- **해결**:
  1. 관리자에게 문의
  2. 일일 한도 초과 여부 확인

## 🚀 빠른 테스트 시나리오

### 시나리오 1: 신규 사용자 전체 플로우
1. 회원가입 → 2. 콘텐츠 3개 생성 → 3. 다음날 복습 수행 → 4. AI 질문 생성 → 5. 대시보드 확인

### 시나리오 2: 구독 업그레이드 플로우
1. FREE 계정 로그인 → 2. 7일 이상 된 복습 확인 (안 보임) → 3. BASIC 업그레이드 → 4. 30일 복습 확인 (보임)

### 시나리오 3: 학습 분석 플로우
1. 일주일간 매일 복습 → 2. 고급 대시보드 확인 → 3. 히트맵 연속 기록 확인 → 4. 주간 목표 달성률 확인

## 💡 테스트 팁

1. **크롬 개발자 도구 활용**
   - Network 탭에서 API 호출 확인
   - Console 탭에서 에러 확인
   - Application 탭에서 localStorage 확인

2. **다중 계정 테스트**
   - 시크릿 모드 사용
   - 다른 브라우저 사용

3. **데이터 초기화**
   ```bash
   docker-compose exec backend python manage.py flush
   docker-compose exec backend python manage.py migrate
   docker-compose exec backend python manage.py create_test_users
   ```

4. **로그 확인**
   ```bash
   # 백엔드 로그
   docker-compose logs -f backend
   
   # 프론트엔드 로그
   docker-compose logs -f frontend
   
   # Celery 로그
   docker-compose logs -f celery
   ```
